## 1. 功能概述

B站视频搜索功能为网易云音乐平台提供了从B站搜索、播放和保存音乐的能力,主要功能包括:

- ✅ **视频搜索**: 通过关键词搜索B站视频,支持分页
- ✅ **音频播放**: 实时解析B站视频并提取音频播放
- ✅ **保存音乐**: 将B站视频音频永久保存到音乐库
- ✅ **搜索历史**: 自动记录搜索历史,支持查询和清空

### 核心特性

- **轻量设计**: 搜索结果仅返回核心字段(bvid、title、duration、url)
- **专注音乐**: 不关注UP主信息和视频统计数据
- **统一管理**: 保存后的音乐与平台音乐统一管理,无来源区分
- **智能提取**: 后端自动提取封面和时长信息,前端无需传递

## 4. 接口详细说明

### 4.1 搜索B站视频

**接口描述**: 根据关键词搜索B站视频,返回分页结果。搜索时会自动记录到搜索历史。

#### 基本信息

- **请求方式**: `POST`
- **接口路径**: `/api/bilibili/search`
- **Content-Type**: `application/json`

#### 请求参数

**请求体 (Body)**:

```json
{
  "keyword": "周杰伦",
  "page": 1,
  "pageSize": 20
}
```

**参数说明**:

| 参数名 | 类型 | 必填 | 默认值 | 说明 | 示例 |
|--------|------|------|--------|------|------|
| keyword | String | 是 | - | 搜索关键词 | 周杰伦 |
| page | Integer | 否 | 1 | 页码(从1开始) | 1 |
| pageSize | Integer | 否 | 20 | 每页大小(最大50) | 20 |

**参数校验规则**:

- `keyword`: 不能为空
- `page`: 最小值为1
- `pageSize`: 最小值为1,最大值为50

#### 响应示例

**成功响应 (200)**:

```json
{
  "code": 200,
  "message": "搜索成功",
  "data": {
    "total": 1500,
    "current": 1,
    "size": 20,
    "pages": 75,
    "records": [
      {
        "bvid": "BV1xx411c7XZ",
        "title": "周杰伦 - 晴天 官方MV",
        "duration": "4:35",
        "url": "https://www.bilibili.com/video/BV1xx411c7XZ"
      }
    ]
  }
}
```

**响应字段说明** (records数组元素):

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| bvid | String | B站视频BV号(唯一标识) | BV1xx411c7XZ |
| title | String | 视频标题(已清理高亮标签) | 周杰伦 - 晴天 官方MV |
| duration | String | 视频时长(格式化字符串) | 4:35 |
| url | String | B站视频完整链接 | https://www.bilibili.com/video/BV1xx411c7XZ |

#### 注意事项

- 搜索时会自动记录到搜索历史表
- 相同关键词重复搜索会增加搜索次数计数
- 标题中的HTML实体已自动解码
- 搜索高亮标签(`<em>`)已自动移除
- 时长格式为 `MM:SS` 或 `HH:MM:SS`

### 4.2 从B站保存音乐

**接口描述**: 将B站视频解析并保存为音乐到音乐库。后端会调用yt-dlp工具提取音频,并自动获取封面和时长信息。

#### 基本信息

- **请求方式**: `POST`
- **接口路径**: `/api/music/from-bilibili`
- **Content-Type**: `application/json`

#### 请求参数

**请求体 (Body)**:

```json
{
  "bvid": "BV1xx411c7XZ",
  "title": "周杰伦 - 晴天",
  "categoryId": 1
}
```

**参数说明**:

| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| bvid | String | 是 | B站视频BV号 | BV1xx411c7XZ |
| title | String | 是 | 音乐标题 | 周杰伦 - 晴天 |
| categoryId | Long | 否 | 音乐分类ID | 1 |

**注意**:

- ❌ 无需传递 `coverUrl` - 由yt-dlp自动提取
- ❌ 无需传递 `duration` - 由yt-dlp自动提取
- ✅ 后端会自动处理封面和时长信息

#### 响应示例

**成功响应 (200)**:

```json
{
  "code": 200,
  "message": "保存成功",
  "data": {
    "musicId": 123,
    "audioUrl": "http://localhost:8910/audio/abc123.mp3"
  }
}
```

#### 注意事项

- 保存操作耗时较长(3-10秒),建议前端显示加载状态
- 音频文件永久保存,不会被定时清理
- 保存后的音乐与平台上传音乐完全一致
- 无来源标识,统一管理在音乐库中

| pageSize | Integer | 否 | 10 | 每页大小 | 10 |

#### 响应示例

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {

    "size": 10,
```

## 6. 错误码说明

### 6.1 业务错误码

| code | message | 说明 | 解决方案 |
|------|---------|------|---------|
| 200 | 操作成功 | 请求成功 | - |
| 400 | 搜索关键词不能为空 | 参数校验失败 | 检查请求参数 |
| 400 | B站视频BV号不能为空 | 参数校验失败 | 检查请求参数 |
| 400 | 页码必须大于0 | 参数校验失败 | 检查分页参数 |
| 400 | 分页大小不能超过50 | 参数校验失败 | 调整pageSize ≤ 50 |
| 500 | 搜索服务暂时不可用 | B站API异常 | 稍后重试 |
| 500 | 视频解析失败 | yt-dlp解析失败 | 检查视频是否可访问或稍后重试 |

## 7. 集成指南

### 7.1 前端集成要点

#### 7.1.1 搜索功能集成

**实现要求**：

1. **输入防抖处理**
   - 在搜索框输入时使用防抖函数,避免频繁请求
   - 建议延迟时间: 500毫秒
   - 防止用户每输入一个字符就发起一次搜索请求

2. **请求超时设置**
   - 搜索接口建议超时时间: 10秒
   - 保存接口建议超时时间: 30秒(需要解析视频)
   - 历史查询接口建议超时时间: 5秒

3. **错误处理**
   - 捕获网络错误并友好提示用户
   - 根据错误码显示不同的错误信息
   - 对于500错误,提示用户稍后重试

4. **加载状态提示**
   - 搜索时显示"正在搜索..."提示
   - 保存音乐时显示"正在解析视频..."等多阶段提示
   - 禁用重复点击,避免重复提交

#### 7.1.2 保存音乐功能集成

**实现要求**：

1. **请求参数准备**
   - 只需传递 `bvid`、`title`、`categoryId` 三个字段
   - 封面和时长由后端自动提取,前端无需处理

2. **长时间等待处理**
   - 保存操作通常需要3-10秒
   - 前端应显示明确的进度提示
   - 建议使用多阶段提示:"解析视频" → "提取音频" → "保存中"

3. **结果反馈**
   - 保存成功后,可跳转到音乐播放页面
   - 失败时,根据错误信息提示用户原因
   - 提供重试选项

#### 7.1.3 历史记录功能集成

**实现要求**：

1. **历史记录显示**
   - 显示最近的搜索历史
   - 按最后搜索时间倒序排列
   - 显示搜索次数统计

2. **点击历史快速搜索**
   - 点击历史记录可快速填充搜索框
   - 自动触发搜索操作

3. **清空历史确认**
   - 清空操作前弹出确认提示
   - 操作不可逆,需提醒用户

### 7.2 状态管理建议

#### 7.2.1 搜索状态管理

**需要管理的状态**：

- `keyword`: 当前搜索关键词
- `videos`: 搜索结果列表
- `loading`: 加载状态
- `error`: 错误信息
- `pagination`: 分页信息(当前页、总页数、总记录数)

**状态更新时机**：

- 用户输入时更新 `keyword`
- 发起搜索时设置 `loading = true`
- 搜索成功时更新 `videos` 和 `pagination`
- 搜索失败时更新 `error`
- 搜索完成时设置 `loading = false`

#### 7.2.2 保存状态管理

**需要管理的状态**：

- `savingVideoId`: 当前正在保存的视频BV号
- `saveProgress`: 保存进度信息
- `savedMusicIds`: 已保存的音乐ID列表

### 7.3 性能优化建议

#### 7.3.1 缓存策略

**搜索结果缓存**：

- 对于相同的搜索关键词和页码,缓存搜索结果
- 缓存时间建议: 5分钟
- 使用LRU策略管理缓存大小(建议最多缓存50个结果)
- 清空历史时同步清空缓存

**历史记录缓存**：

- 缓存最近查询的历史记录
- 缓存时间建议: 1分钟
- 搜索后刷新缓存

#### 7.3.2 分页加载优化

**虚拟滚动**：

- 对于大量搜索结果,使用虚拟滚动技术
- 首次只渲染可视区域的数据(如20条)
- 滚动到底部时加载下一页

**分页切换**：

- 提供"上一页"、"下一页"、"页码跳转"功能
- 记录用户浏览位置,切换页面后保持

#### 7.3.3 并发控制

**批量保存音乐**：

- 限制同时保存的音乐数量(建议最多3个)
- 使用队列管理保存任务
- 显示保存进度和剩余任务数

## 8. 最佳实践

### 8.1 API 调用最佳实践

#### 8.1.1 请求频率控制

**防抖处理**：

- 在搜索框输入时使用防抖函数
- 延迟时间建议: 500毫秒
- 避免每次输入都发起请求

**请求间隔**：

- 搜索接口: 至少间隔500毫秒
- 保存接口: 至少间隔2秒
- 历史查询: 按需调用,不要轮询

#### 8.1.2 超时与重试

**超时设置**：

- 为不同接口设置合理的超时时间
- 搜索接口: 10秒
- 保存接口: 30秒
- 历史查询: 5秒

**重试机制**：

- 对于网络错误,实现自动重试
- 最多重试3次
- 采用指数退避策略(1秒、2秒、4秒)
- 客户端错误(4xx)不重试

#### 8.1.3 错误处理

**统一错误处理**：

- 区分网络错误、超时错误、客户端错误、服务器错误、业务错误
- 为不同类型的错误提供不同的提示信息
- 记录错误日志,便于问题追踪
- 可选:上报错误到监控系统

**用户友好提示**：

- 网络错误: "网络连接失败,请检查网络设置"
- 超时错误: "请求超时,请稍后重试"
- 服务器错误: "服务器错误,请稍后重试"
- 业务错误: 显示后端返回的具体错误信息

---

### 8.2 性能优化建议

#### 8.2.1 缓存策略

**实现方式**：

- 使用内存缓存存储搜索结果
- 缓存键格式: `${keyword}_${page}_${pageSize}`
- 缓存过期时间: 5分钟
- 缓存大小限制: 最多50条记录
- 使用LRU算法管理缓存

**注意事项**：

- 搜索结果缓存5分钟
- 历史记录缓存1分钟
- 不缓存保存操作的响应

#### 8.2.2 分页优化

**虚拟滚动**：

- 首次只渲染可视区域的数据
- 滚动时按需加载更多数据
- 距离底部200px时开始加载下一页
- 使用防抖避免频繁加载

**性能对比**：

- ✅ 虚拟滚动: 首次渲染20条,滚动时按需加载
- ❌ 一次性加载: 首次加载所有数据(可能数百条)

#### 8.2.3 并发控制

**请求队列**：

- 限制同时进行的请求数量
- 建议最多3个并发请求
- 超出限制的请求加入队列等待
- 避免浏览器并发限制和服务器压力

---

### 8.3 用户体验优化

#### 8.3.1 加载状态

**搜索加载**：

- 显示"正在搜索..."提示

## 9. 常见问题

### 9.1 搜索相关问题

**Q1: 搜索时提示"搜索服务暂时不可用"怎么办?**

A: 可能是以下原因:
- B站API临时不可用,请稍后重试
- 网络连接问题,检查网络设置
- 服务器负载过高,等待片刻后重试

**Q2: 搜索结果为空是什么原因?**

A: 可能是以下原因:
- 关键词过于具体,建议使用更通用的关键词
- B站确实没有相关视频
- 建议搜索:"歌手名 + 歌曲名"格式

**Q3: 搜索结果中为什么没有封面图?**

A: v1.3版本为了简化设计,专注音乐本身,移除了封面、UP主等信息。如需查看视频详情,请点击视频链接在B站打开。

---

### 9.2 保存相关问题

**Q4: 为什么保存音乐需要这么长时间?**

A: 保存音乐需要以下步骤:
1. 调用yt-dlp解析B站视频(3-10秒)
2. 提取音频文件
3. 移动到永久存储路径
4. 创建数据库记录

建议在前端显示"正在解析音频,请稍候..."提示。

**Q5: 保存时提示"视频解析失败"怎么办?**

A: 可能是以下原因:
- 视频已被UP主删除或下架
- 视频设置了地区限制
- 视频为付费内容
- 网络不稳定导致下载失败

解决方案:
- 尝试搜索其他相似音乐
- 检查视频在B站是否可正常播放
- 稍后重试

**Q6: 保存后的音乐在哪里查看?**

A: 保存后的音乐会自动添加到音乐库,与平台上传的音乐统一管理,可以通过以下方式查看:
- 音乐列表页面(按创建时间倒序)
- 根据分类筛选(如果保存时选择了分类)
- 通过音乐搜索功能查找

---

### 9.3 技术相关问题

**Q7: API调用频率有限制吗?**

A: 当前版本暂无明确限制,但建议:
- 搜索频率不要过高(建议间隔至少500毫秒)
- 避免短时间内批量保存大量音乐
- 未来可能会增加IP限流和用户限流

**Q8: 接口支持跨域请求吗?**

A: 服务端已配置CORS跨域支持,前端可以直接调用。如遇跨域问题:
- 检查请求头是否正确设置
- 确认服务端CORS配置是否生效
- 考虑使用代理解决开发环境跨域问题

**Q9: 接口是否需要认证?**

A: 当前版本所有接口均无需认证,直接调用即可。未来版本可能会增加用户认证系统,届时需要在请求头中携带Token。
